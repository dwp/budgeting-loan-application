Version 18.1 - lead with email question, then text message as fallback to this, then letter as fallback where neither selected

Ditched - tested in January 2020

  • Pension Credit approach of one thing per page - splitting each contact method out
  • Start by asking if we can contact via email, if yes reveal input box for address
  • If they do not choose email ask if we can contact via text message, if yes reveal input box for mobile phone number
  • If they have provided a mobile number for text messages, ask if we can call them on the number provided - if no ask for alternative number
  • If they do not provide an email address or mobile number for text messages, say we'll send them a letter on the confirmation page
  • Remove sentence that says their reference is their National Insurance number

Start - version 18.1

{#

Version 18.2 - preferred contact method - prioritisation via management information (MI)

  • Prioritisation via research and MI on prefered way to be contacted/cost/failure of delivery
  • User can only select one
  • Ask them for email first as it was the highest of the three in preferred contact method research/doesn't cost to send
  • If user selects yes, reveal input for address
  • if not email then ask for phone number to text, second preference method/on average costs £7pw on failed to send texts
  • If user selects yes, reveal input for number
  • Third option is letter if user really doesn't want to be contacted this way (do we need to reveal text that explains this is slower?)

Start - version 18.2
